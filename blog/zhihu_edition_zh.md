# ROLL 0.2：局部分时调度如何榨干 GPU 算力，终结 Agentic RL 的资源泡沫

> **一句话总结：** Roll 0.2 的 **局部分时（Partial Time-Sharing）** 调度让 GPU 在采样阶段也能跑训练，把「空等长尾样本」的时间变成「有效训练时间」。在阿里巴巴生产环境的 160 卡集群上实测：**rollout 吞吐提升约 3 倍，采样超时率从 90% 降到 0**。更重要的是，**与标准异步训练语义等价，训练配置无需任何修改**。
>
> **[📂 查看 GitHub](https://github.com/alibaba/ROLL)**

## 为什么 GPU 总在空转？

大模型正在从"只会聊天"变成"能干活"的智能体（Agent），但这也给分布式 RL 训练带来了新麻烦，**落后者效应（straggler effect）**。

简单说就是：rollout 采样的耗时呈**长尾分布**。大部分样本任务较快搞定，但总有几个"落后者"要跑很久。**大部分 GPU 干等着最后几个样本，这段时间就是纯粹的算力浪费**，我们叫它"资源泡沫"（Resource Bubble）。

Agentic RL 把这个痛点放大了三倍：
1. **回合数不确定**：有的任务 3 轮对话解决，有的要 100 轮，完全看命
2. **输出长度飘忽**：推理深度时深时浅，token 数差一个数量级
3. **外部工具拖后腿**：调编译器、查 API，响应时间全看对方心情

传统调度策略面对这种不确定性，基本就是"硬扛"，资源浪费严重。

---

## 三种调度方案，谁更香？

对比业界主流的两种做法和ROLL的局部分时调度：

![三种调度策略时间线对比](timeline%20compare%20zh.png)
*图1：三种方案的时间线对比。局部分时的关键在于"见缝插针"，采样凑齐训练的 batch 后，释放 GPU 切去训练。*

| 方案 | 核心思路 | GPU利用率 |
|:---|:---|:---|
| **同步共置** | 采样→训练→采样，严格串行 | 所有 GPU 等最慢的样本，利用率感人 |
| **异步分离** | 128 卡固定训练，32 卡固定采样 | 两边各自等长尾，"双重泡沫" |
| **局部分时** | 采样凑齐训练的 batch 后，剩余采样任务较少，释放 GPU 切去训练 | **泡沫减少，提升资源利用率** |

## 具体怎么实现？

思路很简单：**训练的batch凑齐了，就把部分rollout的 GPU 切去跑训练。**

整个流程分两步走：
<!-- todo psuedocode -->
**阶段一：全员采样**
- 160 卡全部投入 rollout，最大并行度清掉积压的采样任务

**阶段二：并行执行训练和采样**
- **Rollout缩容**：攒够一个 batch 的数据后，一部分 GPU 切权重去训练，剩下的继续啃长尾样本
- **并行跑**：训练和采样同时跑，互不耽误
- **Rollout扩容**：训练完一轮，所有节点同步最新权重，切回全员采样模式

## 性能测试：生产级大规模测试

### 实验配置

| 参数 | 配置 |
|:---|:---|
| **总 GPU 数** | 160 张卡 |
| **基线（异步分离）** | 128 卡训练 + 32 卡采样 |
| **局部分时** | 128 卡固定训练，160 卡动态采样（训练卡可复用） |
| **模型** | Qwen3-235B-A22B（MoE，235B 总参 / 22B 激活） |
| **任务** | SWE-Agentic（100 轮交互，64K token，batch=512） |
| **异步比（轨迹陈旧度约束）** | 1 |

### 关键结果

| 指标 | 基线（异步分离） | 局部分时 | 提升 |
|:---|:---|:---|:---|
| **相对每步训练延迟** | 0.6 | **0.2** | **3倍吞吐提升** |
| **采样超时率** | 90% | **≈0%** | **几乎零浪费** |
| **权重同步开销** | <1 分钟 | **多几秒** | **可忽略** |

两个核心收益：

1. **吞吐提升 3 倍**：160 卡全上采样，batch 凑齐速度快了，训练不用干等，相对延迟从 0.6 降到 0.2

2. **超时归零**：采样完成得早，不再触发超时阈值，样本利用率从 10% 提升到 100%

![训练步延迟对比](./eval%20zh%20rollout%20time.jpg)
*图2：训练步延迟对比。0.6→0.2 的差距，就是被消灭的"资源泡沫"。*

![采样超时率对比](./eval%20zh%20timeout.jpg)

## 总结与展望

**局部分时把"等 GPU"变成"用 GPU"**，长尾样本不再是资源杀手，而是训练并行的时间窗口。3 倍吞吐、零超时、零配置上手。

ROLL 独立地提出并在大规模生产集群和大尺寸 MoE 模型上验证了局部分时调度方法，并将经过阿里巴巴内部实战检验的工业级方案**完全开源**。感谢 **[SeamlessFlow](https://arxiv.org/abs/2508.11553v1)** 分享 Tag Scheduling 工作，他们的工作也证明了这一调度思路的有效性。

**Multi-LoRA RL 即将开源！** 受 [Tinker API](https://thinkingmachines.ai/tinker) 启发，我们已将局部分时调度拓展到多 LoRA 并行训练场景，一个 RL 任务同时训多个适配器，效率更上一层。
